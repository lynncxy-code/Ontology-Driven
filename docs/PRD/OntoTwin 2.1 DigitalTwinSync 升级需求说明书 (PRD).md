## 1. 项目愿景

从“点对点”的手动三维映射，进化为基于**本体能力接口（Capability Interface）**的自动化驱动体系，通过“元模型”层面的约束，确保三维表现与业务逻辑的严谨对齐。

## 2. 核心架构逻辑

2.0 版本引入**“接口抽象层”**，实现逻辑层与表现层的深度解耦：

- **本体层**：定义业务对象及其属性节点（物理库映射完成）。
    
- **接口层（新增）**：定义三维能力契约（如可移动、可变色）。
    
- **映射层**：定义属性到接口的 Function 转换关系。
    
- **执行层**：UE 插件根据接口契约动态驱动资产行为。

## 3. 核心功能需求

### 3.1 三维能力接口治理 (Capability Governance)

**目标**：将三维表达属性标准化为可继承的“接口”。

- **接口注册库**：系统预设标准三维接口，包括但不限于：
    
    - `I3D_Representable`：包含【文件编号】（关联模型库）、LOD 精度、显隐状态。
        
    - `I3D_Movable`：包含世界坐标 $X, Y, Z$。
        
    - `I3D_Rotatable`：包含欧拉角或四元数旋转。
        
    - `I3D_Animatable`：包含动画名称、进度、循环策略。
        
- **能力注入（Implement）**：支持在 ObjectType 定义阶段，勾选其支持的接口。
    
    - _示例：为“飞机”类对象勾选 `I3D_Movable`，则该类所有实例自动获得位移属性槽。_
        

### 3.2 可视化映射控制台 2.0 (Mapping Console)

**目标**：通过连线和函数定义，打通本体属性与接口属性的“最后一公里”。

- **左右连线界面**：
    
    - 左侧显示本体属性节点（支持查看物理表/字段溯源信息）。
        
    - 右侧显示已绑定的三维接口属性槽。
        
- **中间函数处理器（Function Handler）**：
    
    - 支持在连线中间插入计算节点。
        
    - 预设函数：线性缩放（Unit Conversion）、枚举映射（Enum Map）、数值过滤（Deadband）。
        
- **资产绑定（Asset Binding）**：
    
    - 对接“模型资产库”。
        
    - 通过【文件编号】为接口 `I3D_Representable` 绑定对应的三维模型（优先关联二级优化区文件）。
        

### 3.3 实例管理双层清单 (Instance Management)

**目标**：建立本体清单与实例清单的级联视图。

- **ObjectType 层级（类目层）**：展示所有实现（Implement）了三维接口的对象类（如工装、AGV、飞机零件）。
    
- **Instance 层级（实例层）**：
    
    - 展示具体实例清单，包括 ID、在线状态、数据延迟。
        
    - **实时快照查看**：直接在列表中查看该实例当前从物理库同步过来的原始值与转换后的表现值。
        

### 3.4 双向数据校验与约束 (Data Validation)

**目标**：解决本体定义与三维资产“虚实不符”的痛点。

- **物理参数校验**：
    
    - 自动比对模型资产库中的元数据（如：模型的包围盒尺寸）与本体定义的物理规格（如：零件长度）。
        
    - 若偏差超过阈值，在实例管理界面标红预警。
        
- **能力可用性校验**：
    
    - 若本体开启了“旋转接口”，但关联的 3D 资产（模型）不支持旋转，系统提示“模型能力缺失”。

## 4. 关键技术指标 (KPI)

- **配置效率**：单类资产（ObjectType）的 3D 接口配置时间 $\leq$ 10 分钟。
    
- **动态加载**：UE 插件根据 `asset_id` 自动从模型库拉取资产的成功率 $\geq$ 99.9%。
    
- **同步延迟**：从物理库数据更新到 3D 画面响应（含后端 Function 计算时间） P95 $\leq$ 1s。

## 5. 用户用例 (Use Case) 示例

**用例：工装尺寸异常预警**

1. **定义阶段**：在 Mapping Console 为“工装类”绑定 `I3D_Representable` 接口，关联模型库中的【文件编号】。
    
2. **映射阶段**：将本体属性 `actual_length` 连线至接口的校验字段。
    
3. **运行阶段**：后台自动比对模型库中该编号资产的真实三角面顶点距离与 `actual_length`。
    
4. **反馈阶段**：若数据不符，三维场景中该工装模型闪烁报警，网页实例清单标红。
