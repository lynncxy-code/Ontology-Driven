## 一、产品定义与核心价值

- **定位**：工业级"本体驱动"三维配置与运维平台，实现从业务语义到三维物理表现的自动化闭环。

- **核心理念**：数据即真相。三维场景不再是手动摆放的静态画卷，而是由本体实例动态生成的"运行态"镜像。

- **2.3 范围说明**：接口规范层（V1.0）保持不变；业务属性与接口属性的映射关系由系统内置，采用 1:1 直通方式，后端预配置，用户无需操作；资产格式统一为 `.glb`。

## 二、系统架构

### 2.1 总体架构

```
┌─────────────────────────────────────────────┐
│           数据接入层 (Data Ingestion)          │
│   Web 控制台 — 本体配置 / 实例管理 / 监控       │
└───────────────────┬─────────────────────────┘
                    │ HTTP / JSON
┌───────────────────▼─────────────────────────┐
│           本体服务层 (Twin Core)               │
│   Python 后端 — 权威状态存储 / 映射计算 / 广播   │
└───────────────────┬─────────────────────────┘
                    │ HTTP / JSON（轮询 500ms）
┌───────────────────▼─────────────────────────┐
│           3D 渲染层 (3D Runtime)              │
│    3D 渲染端 — 加载 GLB 资产 / 驱动场景      │
└─────────────────────────────────────────────┘
```

### 2.2 技术栈

| 层级 | 技术选型 |
|---|---|
| **后端** | Python 3.10+，Flask（RESTful API） |
| **前端控制台** | HTML5，JavaScript，Vue.js |
| **3D 渲染端** |  3D 渲染环境（前端页面orUE 引擎） |
| **通信协议** | HTTP / JSON，Polling 模式，间隔 500ms |
| **3D 资产格式** | `.glb`（GLB 二进制格式） |

### 2.3 数据流向

1. **配置操作**：用户在控制台为 ObjectType 注入接口、绑定 GLB 资产。
2. **实例投产**：用户新建实例，后端创建实例记录，触发 3D 渲染端加载资产。
3. **状态写入**：物理数据或手动 Override 通过 POST 更新后端实例属性。
4. **状态分发**：3D 渲染端每 500ms 轮询后端，获取最新表现值。
5. **驱动渲染**：3D 渲染端根据接口属性更新场景对象的位置、材质、动画等。

## 三、核心接口规范（V1.1）

> 系统级契约，与渲染端实现无关。采用**两层结构**：`I3D_Representable` 为顶层能力标记，是对象在三维世界存在的必要前提；三个子能力接口在此基础上按需挂载。

```
I3D_Representable  （顶层 · 必选 · 存在性声明）
├── I3D_Spatial    （子能力 · 可选 · 空间变换）
├── I3D_Visual     （子能力 · 可选 · 视觉表达）
└── I3D_Behavioral （子能力 · 可选 · 动态行为）
```

### 3.1 I3D_Representable — 顶层能力标记接口

**语义**：声明"该对象具备在三维场景中存在的能力"。所有需要在场景中呈现的对象必须首先实现此接口，三个子能力以此为前提。

| 属性标识（RID） | 名称 | 数据类型 | 描述 | 备注 |
|---|---|---|---|---|
| `asset_id` | 资产资源路径 | String | 引擎端直接可加载的物理资源路径（如 `/Game/Models/...`） | 由后端根据 FileNumber 自动翻译下发 |
| `file_number` | 资产业务编号 | String | 关联资产库或 PLM 系统中的逻辑模型编号 | 用户在控制台绑定的原始值 |
| `is_visible` | 是否渲染 | Boolean | 控制对象整体的场景加载与显隐 | `false` 时对象不出现在场景，默认：`true` |

### 3.2 I3D_Spatial — 空间变换接口（子能力）

**语义**：赋予对象在三维空间中定位、旋转、缩放的能力。

| 属性标识（RID） | 名称 | 数据类型 | 描述 | 备注 |
|---|---|---|---|---|
| `translation_x/y/z` | 位置 | Float | 物体世界坐标 | 单位：厘米（cm），现实 1:1 |
| `rotation_x/y/z` | 旋转 | Float | 物体绕轴旋转角度 | 单位：度（0–360） |
| `scale_x/y/z` | 缩放 | Float | 三轴缩放倍数 | 默认值：1.0 |

### 3.3 I3D_Visual — 视觉表达接口（子能力）

**语义**：赋予对象切换材质外观的能力。资产加载与整体显隐由父接口 `I3D_Representable` 统一管理。

| 属性标识（RID） | 名称 | 数据类型 | 描述 | 备注 |
|---|---|---|---|---|
| `material_variant` | 材质变体 | String | 触发材质切换的状态键 | 如：`normal`、`alarm`、`offline` |

### 3.4 I3D_Behavioral — 动态行为接口（子能力）

**语义**：赋予对象执行动态行为与信息标注的能力。

| 属性标识（RID） | 名称 | 数据类型 | 描述 | 备注 |
|---|---|---|---|---|
| `animation_state` | 动画状态 | String | 驱动动画的状态名称 | 如：`idle`、`running`、`broken` |
| `fx_trigger` | 特效触发 | String | 触发粒子或后期特效的名称 | 如：`spark`、`smoke` |
| `ui_label_content` | 标签内容 | String | 悬浮于物体上方的 UI 文本 | 展示实时业务数值 |

## 四、系统映射契约（内置规范）

> 本章说明系统如何将业务属性值转换为接口表现值。此过程由后端自动执行，用户无需配置操作界面。

- **映射策略（2.3 版本）**：为了实现最极致的性能与开发效率，当前 V2.3 版本的业务属性与底层接口属性采用 **1:1 绝对直通（Passthrough/Identity）** 映射。即无论坐标轴、比例、还是材质名（`normal`、`alert` 等），都要求上游物理业务系统在推送数据时已经符合引擎端的标准。
- **边界划分**：本系统在 V2.3 阶段不承担任何“数据清洗翻译”工作（不提供内置枚举转换或线性换算表）。一切计算压力均外包给物理系统的前置网关或数据平台。
- **扩展性**：后续版本可在此基础上将状态映射下沉到平台内，引入 `Linear_Remap`、`Enum_Map` 等后台可配置算子，本文档附录 A 已为此预留概念雏形。

## 模块 1：本体能力注入中心（Ontology Definition & Injection）

**目标**：在元模型层面为业务对象赋予"三维能力"，并通过挂载 vs 不挂载接口的对比，直观展示 3D 渲染端的响应差异。

### 1.1 本体浏览器

- 从后端拉取 `ObjectType` 列表（如：工装、AGV、飞机零件）。
- 展示每个对象的属性定义（属性名、标识、数据类型）及当前已挂载的接口状态。
- 支持区分展示：**已具备 3D 能力**（已挂载接口）vs **未具备 3D 能力**（未挂载），两类对象在控制台有视觉区分。

### 1.2 能力接口挂载（Interface Injection）

- 接口采用两层结构，按以下顺序挂载：
  - **第一层（必选）** `I3D_Representable`：顶层能力标记，声明该对象具备三维存在性；挂载后须在 1.3 节完成 GLB 资产绑定。
  - **第二层（可选，按需组合）**：
    - `I3D_Spatial`：空间变换能力（位置、旋转、缩放）。
    - `I3D_Visual`：视觉表达能力（材质变体切换）。
    - `I3D_Behavioral`：动态行为能力（动画、特效、信息标签）。
- **挂载依赖边界约束**：
  - **前置限制**：未挂载 `I3D_Representable` （声明基础实体缺失）时，系统从接口层面拒绝该 ObjectType 的任何子特征（Spatial/Visual/Behavioral）被单独挂载。
  - **级联卸载**：若从对象上主动卸载 `I3D_Representable`，系统将强制进行**级联清空**，一并销毁附属于它的所有空间、视觉、行为子表现能力。
- **配置态演示价值**：对同一 ObjectType 逐步挂载子能力，可实时观察 Web 端控制面板从"空控制块"到逐步膨胀出"定位"、"调色"、"拨弄动画"表单域的过程。

### 1.3 资产预绑定

- 实现 `I3D_Representable` 时，填写 **FileNumber（文件编号）** 以关联资产库中的 `.glb` 模型。
- 系统自动校验该 FileNumber 在资产库中的有效性（文件存在性、格式正确性）。
- 绑定关系持久化至后端，供 3D 渲染端动态加载使用。

## 模块 2：实例运维与监控中心（Instance Ops & Monitoring）

**目标**：管理孪生体的生命周期，提供实时状态监控与简单的状态指示。

### 2.1 实例投产（Spawn）

- 基于已完成接口挂载的 `ObjectType` 新建实例，系统分配唯一 `InstanceId`（如：`plane_002`）。
- 支持填写初始空间参数（位置 X/Y/Z），提交后：
  - 后端创建实例记录。
  - 3D 渲染端接收到新实例后，从资产库拉取对应 `.glb` 文件并在场景中创建对象。
- 实例清单展示：ID、ObjectType、在线状态、最后同步时间。

### 2.2 实时监控面板

- 选中任意实例，展示其当前状态数据：
  - **原始值**：从物理数据库同步过来的业务属性值。
  - **表现值**：经后端处理后下发给 3D 渲染端的接口属性值。
- **手动 Override**：调试模式下，通过输入框直接改写表现值，实时观察 3D 渲染端响应。
- 数值以接口分组展示（I3D_Spatial / I3D_Visual / I3D_Behavioral）。

### 2.3 实例状态指示

- 每个实例在清单中显示简单状态标识：
  - 🟢 **在线**：数据正常同步。
  - 🔴 **离线**：超过 3.0s 未收到数据更新（内置心跳容忍期）。
  - 🟡 **预警**：后端检测到非法值被钳位，或资产加载失败。
- 点击预警实例可查看简要原因描述。

# Part B：3D 渲染端实现规范

> 面向 3D 渲染端开发者。定义渲染端与系统其他层的集成契约，具体渲染实现由开发者自行决策。

---

## B.1 资产动态加载

- 在我们目前的 2.3 最新实现中，为了最大化引擎性能并取消 HTTP GLB 下载的开销，后端下发的快照里 `interfaces.I3D_Representable.asset_id` 已经**直接翻译为 UE 引擎的本地资产路径**（例如 `/Game/WarehouseProps/SM_Pallet_01a`）。
- 渲染端只要拿到 `asset_id`，直接调用 `LoadObject<UStaticMesh>` 进行资源加载即可，无需再发额外的网络请求。
- 系统内的 `file_number` 仅存在于 Web 绑定阶段和后端的映射代理层，对渲染端透明。
- 若资产加载失败，引擎渲染侧需能够自我诊断，并通过向控制台调用 `POST /api/v2/state/feedback` 接口主动抛出 `asset_load_error` 标识及堆栈以供服务端运维预警。

## B.2 状态驱动（接口执行映射）

- **实例发现与全量驱动**：渲染端*无需额外实现任何长连接注册协议*。以 **500ms** 为间隔，持续轮询 `GET /api/v2/instances`（该接口即刻返回当前网格内生存的**全量实例快照数组**）。引擎只要遇缺则 Spawn（新实例创建），寻不到则 Destroy（实例彻底注销）。
- **非挂起轮询 fallback**：如果在极端需求下针对特定实体寻更短周期的观察，支持发起 `/api/v2/state/snapshot?id={instanceId}` 针尖调用。
- **静态属性传输说明**：由于 2.3 版本遵循完全无状态设计，快照负荷中会持续携带 `asset_id` 等静态属性。这虽然轻微增加了每 500ms 几十字节的带宽，但使得 UE 客户端无需自行维护复杂的状态机（网络闪断重连后即可秒级恢复渲染），属于典型的“以带宽换架构稳定性”的 Trade-off。
- 根据返回的接口属性，驱动对应场景对象：

| 接口 | 层级 | 渲染端响应行为 |
|---|---|---|
| `I3D_Representable` | 顶层·必选 | 根据 `asset_id` 加载 GLB 资产并创建场景对象；`is_visible = false` 时卸载对象，不占渲染资源 |
| `I3D_Spatial` | 子能力·可选 | 更新对象在场景中的 position / rotation / scale |
| `I3D_Visual` | 子能力·可选 | 根据 `material_variant` 切换材质外观 |
| `I3D_Behavioral` | 子能力·可选 | 切换动画状态、触发特效、更新浮层标签文本 |

> **`I3D_Representable.is_visible` 说明**：控制对象是否**存在于场景**（`false` 时对象从场景卸载，不占用内存与渲染资源）。这与材质层面的"隐藏"不同——后者对象仍在场景中，只是不被渲染。

- **接口能力隔离**：未挂载某接口时，渲染端忽略对应属性，不执行该类操作。未挂载 `I3D_Representable` 的实例，渲染端不创建任何场景对象。

## B.3 异常处理

| 场景 | 渲染端行为 |
|---|---|
| 超过 3.0s 未收到新的有效快照 | 将实例切换至"离线"视觉状态（如灰色材质） |
| 返回值包含非法数值（负缩放等） | 本地二次 Clamp：<br>1. **Scale**: 下限钳位至 `0.001`，禁止负值或完全为 0 导致渲染崩溃。<br>2. **Rotation**: 对 `360.0` 取模，防止浮点溢出。<br>3. **Translation**: 无引擎级限制，但突变时建议应用插值（Interp）。 |
| 实例被删除 | 通过全量数组比对判定缺失，从场景中安全移除对象，释放内存资源 |

## B.4 UE 侧本地干预机制（编辑器提效）

为适应 UE5 地图美术与关卡设计的本地测试需求，渲染侧组件需额外支持以下两项本地脱机干预能力：

### 1. 锁定本地空间变换 (Local Override Lock)
- **场景痛点**：由于引擎每 500ms 会强制被后端数据“复位”，当美术人员或开发者在引擎 Editor 内试图拖拽某个实例以预览不同位置的效果时，对象会立刻“瞬移”回原位，无法进行本地级别的空间调试。
- **机制要求**：`TwinInstance` 需向面板抛出一个布尔值开关（如 `bLocalOverrideLock`）。开启该锁后，实例在接收并在内部刷新状态时，**主动剥离/忽略 `I3D_Spatial` （平移、旋转、缩放）组件的强覆盖**，使得开发者在此期间可以像操作普通 Actor 一样自由摆位。

### 2. 快照固化到关卡 (Snapshot Baking/Freezing)
- **场景痛点**：通过 500ms 短连接 Spawn 出来的孪生体均为动态临时 Actor，当结束 PIE（Play In Editor）运行后，这些数据随之飞灰湮灭。这意味着它们无法被长久地留在场景文件中参与传统游戏引擎的离线静态光照烘焙和导航网格（NavMesh）预构建。
- **机制要求**：赋予 `DigitalTwinSyncComponent` 一个能够在 Editor 态执行的“一键烘焙”（Bake）能力。该功能生效时，将当前网络快照中的所有实例及其外挂材质变体，“斩断”数字孪生同步流链路，无缝转化为标准的、不再随后端刷新的底层 `StaticMeshActor`，以供传统美术流进行后续地图包装。

## 五、后端 API 契约

### 5.1 接口一览

| 接口路径 | 方法 | 描述 |
|---|---|---|
| `/api/v2/ontology/types` | GET | 获取 ObjectType 列表及属性定义 |
| `/api/v2/ontology/interfaces` | GET | 获取全部能力接口定义 |
| `/api/v2/ontology/inject` | POST | 为 ObjectType 注入能力接口 |
| `/api/v2/ontology/inject/remove` | POST | 从 ObjectType 卸载某能力接口（UI 上的“修改”模式） |
| `/api/v2/assets/bind` | POST | 绑定 FileNumber 与 ObjectType |
| `/api/v2/assets/resolve` | GET | 通过 `file_number` 获取 GLB 资产下载地址 |
| `/api/v2/instances` | GET | 获取全量实例清单及快照表现值（返回数组，引擎批量轮询端点） |
| `/api/v2/instances` | POST | 创建新实例（投产） |
| `/api/v2/instances/<id>` | DELETE | 销毁与注销指定实例 |
| `/api/v2/state/snapshot` | GET | 获取指定实例的表现值快照 |
| `/api/v2/state/override` | POST | 手动干预 Override（警告：无状态锁定，会在下一次 Ingest 时立刻被外部物理数据无情覆盖） |
| `/api/v2/state/ingest` | POST | **核心数采接入点**：外部 SCADA 系统持续推送直通物理属性（如 `translation_x`）至本项目 |
| `/api/v2/state/feedback` | POST | **渲染端上行通道**：供 UE5/Web3D 向后端自报家门状态异常（如资产 404、内存爆出） |

## 七、关键技术指标（KPI）

| 指标 | 目标值 |
|---|---|
| **端到端延迟** | 从数据更新到 3D 画面响应，P95 ≤ 1s |
