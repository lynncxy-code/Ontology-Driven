## 一、背景与动机

### 1.1 现有痛点

"实例投产"当前只支持**单个逐一操作**，用户每次只能投产一个实例，在面对以下场景时效率极低：

- 从企业 MES/SAP 等外部系统同步了一批已知实例（如 `AGV-001` 至 `AGV-010`），需要为它们批量配置场景坐标后一次性投入 UE 场景。
- 同类型设备在工厂中大量并存，用户希望一次操作完成"选实例 → 配坐标 → 投产"。

### 1.2 核心设计洞察

> **批量投产的本质，不是批量创造实例，而是批量为"已知实例"分配场景坐标。**

在真实企业场景中，实例 ID（如 `AGV-001`）通常来源于外部权威数据库（MES/SAP），已通过 `POST /api/v2/instances` 提前注册到孪生平台。用户并不需要"发明新 ID"，而是需要从平台中**选择尚未部署到场景的现有实例**，然后为每一个手动指定初始坐标。

## 二、功能需求

### 2.1 触发入口

在"实例运维与监控中心"的实例清单顶部，新增 **"批量配置部署"** 按钮，打开批量配置抽屉（Drawer）或弹窗。

### 2.2 批量配置界面

批量配置界面分为两步：

---

#### 第一步：选择实例

展示一个**可多选的实例列表**，仅列出满足以下条件的实例：
- 已在后端注册（`GET /api/v2/instances` 中存在）
- **当前没有有效的空间坐标**（`translation_x/y/z` 全部为 0 或为空）

> [!NOTE]
> 过滤掉"已部署"的实例，避免用户意外覆盖已有坐标。

用户可通过 `ObjectType` 筛选只看特定设备类型（如只看 AGV），然后勾选目标实例。

---

#### 第二步：逐一配置坐标

选完后，界面展示一张**可编辑的表格**，每行对应一个已选实例：

| 实例 ID | ObjectType | X (cm) | Y (cm) | Z (cm) | 旋转 Z (°) |
|---|---|---|---|---|---|
| AGV-001 | AGV | `[输入框]` | `[输入框]` | `[输入框]` | `[输入框]` |
| AGV-002 | AGV | `[输入框]` | `[输入框]` | `[输入框]` | `[输入框]` |
| AGV-003 | AGV | `[输入框]` | `[输入框]` | `[输入框]` | `[输入框]` |

- 坐标初始值默认为 `0`，用户手动修改每个实例的位置。
- 旋转只暴露最常用的 Z 轴（偏航角，控制朝向），其余 X/Y 旋转默认为 0（可在单实例详情页单独调整）。
- 支持**键盘 Tab 键横向跳格**，提高输入效率。

---

### 2.3 提交与部署

点击 **"确认部署"** 按钮后：

1. 前端校验：每个实例至少填写了 X/Y/Z（不允许全部为 0，避免叠堆）。
2. 对每个实例，调用 `POST /api/v2/state/override`，下发新的 `I3D_Spatial` 坐标。
3. 显示进度条（`3/10 正在部署…`），全部完成后显示摘要。
4. UE 端在下一次 500ms 轮询中自动读取新坐标，将实例移至正确位置。

> [!IMPORTANT]
> 此处使用 `override` 接口而非重新 `POST` 创建，因为实例本身已存在，我们只是在更新它的坐标。

---

## 三、后端 API 使用方式

| 操作 | 接口 | 说明 |
|---|---|---|
| 拉取候选实例列表 | `GET /api/v2/instances` | 前端过滤出无坐标的实例 |
| 下发坐标 | `POST /api/v2/state/override` | 为每个选中实例写入 `I3D_Spatial` |

**本次不新增任何后端接口。**

---

## 四、UX 交互流程

```
点击"批量配置部署"
  └─ 抽屉展开：显示"未部署实例"列表
       └─ 用 ObjectType 筛选 → 勾选目标实例 → 点"下一步"
            └─ 进入坐标配置表格
                 ├─ 逐行手动输入 X / Y / Z / 旋转Z
                 └─ 点击"确认部署"
                      ├─ 校验坐标不全为 0
                      ├─ 串行 override × N，显示进度
                      └─ 完成 → 摘要弹窗
                               └─ UE 端 500ms 内自动感知坐标变更
```

## 五、验收标准

| 场景 | 通过条件 |
|---|---|
| 正常批量部署 5 个 AGV | 5 个实例在 UE 场景中出现在各自填写的坐标位置 |
| 已部署实例不在候选列表 | 已有坐标的实例默认不出现在批量选择列表中 |
| 坐标全为 0 时提交 | 前端拦截，提示"请为每个实例填写有效坐标" |
| 部分失败 | 已成功的实例不回滚，失败实例在摘要中列出 |
| 长列表下的可用性 | 50 个候选实例时，列表可搜索+分页，不卡顿 |

---

## 六、范围外（Out of Scope）

- **自动坐标计算**（间距/排列轴）：坐标由用户手动输入，系统不做推算。
- **批量创建新实例**：本功能仅服务于"已在系统中注册的实例"，新实例创建仍使用现有单个投产流程。
- **批量销毁实例**：列为后续版本功能。
- **UE 端改动**：无需任何修改。
