# OntoTwin 2.9.2.2 图层选择性纳入需求说明书 (PRD)

> **定位**：在 2.9 坐标标定工作台 CAD 模式的第①步（上传），为每个图层增加一个独立于"画布显隐"的**「选择/纳入」开关**。未勾选的图层，其全部内容在后续所有环节（类型审核、标定、实体表、导出、投实例）一律不出现。
>
> **父需求**：2.9.2 CAD 本体类型自动建库 / 2.9 坐标标定工作台。

---

## 1. 背景与问题

当前第①步的图层列表只有一个「眼睛」开关（`layerVisible`），它实际上**身兼两职**：

| 现状职责 | 作用范围 |
| :--- | :--- |
| 画布显隐 | `draw()` 是否绘制该图层的几何 |
| 流程纳入（隐性） | 实体表（step4）、导出（step5）、投实例（2.9.1）都用它做过滤 |

带来两个问题：

1. **概念混淆**：用户以为眼睛只是"画布上看不看得见"，但它其实也决定了"这个图层会不会被导出/投产"。语义不清。
2. **审核环节漏过滤**：**类型审核（step2）没有按图层过滤**——隐藏了某图层，画布/实体表里它消失了，但 step2 的候选类型列表里照样出现。前后不一致。

**用户诉求**：把"看不看得见"和"要不要这个图层"拆成两个开关。后者（选择）一旦取消，该图层在**所有环节彻底消失**。

---

## 2. 核心设计：两个独立开关

| 开关 | 标识 | 默认 | 作用 |
| :--- | :--- | :--- | :--- |
| **选择 / 纳入** | 复选框 ☑（新增） | 全选 | 该图层是否进入整条流水线（审核/标定/实体/导出/投实例）。**取消 = 该图层在后续全部环节消失。** |
| **可见性** | 眼睛 👁（现有） | 全显 | 仅控制当前画布上是否绘制。纯视觉，便于在密集图纸里临时减少干扰。 |

### 2.1 两者关系：**不变式「选择 ⟹ 可见」**（用户拍板）

| 选择 | 可见 | 合法 | 含义 |
| :--: | :--: | :--: | :--- |
| ✓ | ✓ | ✅ | 正常：纳入流程 + 画布显示 |
| ✓ | ✗ | ❌ | **非法**——选中的图层必须可见 |
| ✗ | ✓ | ✅ | 排除出流程，但留在画布上供参考（"看得见但不纳入"）|
| ✗ | ✗ | ✅ | 排除 + 隐藏 |

**交互推论**：
- **勾选「选择」** → 同时置 `可见=true`，且眼睛**锁定为开**（选中的图层不可隐藏；要隐藏须先取消选择）。
- **取消「选择」** → 眼睛**解锁**，可自由开关；可见性保持当前值（默认仍可见，即"取消后仍看得见"，用户可再手动隐藏）。
- **眼睛开关** → 仅在「未选择」时可用；「已选择」时显示为锁定的常亮态（带提示文案）。

### 2.2 职责重新划分（两者正交）

引入 `layerSelected` 后，两个开关**互不叠加、各管一摊**：

| 环节 | 改造前过滤依据 | 改造后过滤依据 |
| :--- | :--- | :--- |
| 画布 `draw()` | `layerVisible` | `layerVisible`（不变；因"选择⟹可见"，选中的天然会画）|
| 类型审核 step2 | （无过滤）❌ | `layerSelected` ✅ |
| 实体表 step4 | `layerVisible` | `layerSelected` |
| 导出 step5 | `layerVisible` | `layerSelected` |
| 投实例 2.9.1 | `layerVisible` | `layerSelected` |

> 即：**流水线只看「选择」，画布只看「可见」**。不变式「选择⟹可见」由交互层（勾选时强制置可见、锁眼睛）保证。

---

## 3. 影响范围（逐环节）

| 环节 | 未选图层的表现 |
| :--- | :--- |
| ① 上传 | 图层行复选框取消；该行置灰；眼睛禁用；画布移除该图层几何 |
| ② 类型审核 | 该图层下的所有候选块（按 `primary_layer` 判定）不出现在审核列表；摘要计数相应减少 |
| ③ 标定 | 画布不绘制该图层；锚点标定不受影响（锚点是用户手点的世界坐标，与图层无关）|
| ④ 实体 | 实体表不列出该图层的 INSERT / POLYLINE |
| ⑤ 导出 | 导出 JSON / CSV 不含该图层实体 |
| 投实例（2.9.1）| 不投产该图层的实体 |

---

## 4. 边界情况（请你一并考虑）

### 4.1 块横跨多个图层
扫描结果里，一个 `block_name` 可能出现在多个图层（`candidate.layers` 是数组，`primary_layer` 是出现最多的那个）。
- **场景**：块 X 在「选中图层 A」出现 50 次、在「未选图层 B」出现 3 次。
- **决策点**：按 `primary_layer`（主图层）判定，还是"只要任一图层被选中就保留"？详见 §5-决策2。

### 4.2 墙体/地面（POLYLINE）
它们也按图层归属，受选择开关影响。取消某墙体图层 → 导出 JSON 里不含该墙。符合预期。

### 4.3 与 step2 审核已有勾选（`auditChecked`）的叠加
形成**两级闸门**，互不冲突：
- **图层选择（step1）= 上游粗筛**：决定哪些图层的块能进入审核列表。
- **审核勾选（step2）= 下游精选**：在已进入的候选里，再选哪些真正建为类型。
- 取消图层选择 → 该图层的块直接不进审核列表，连被勾选的机会都没有。

### 4.4 现有「眼睛」行为变更
眼睛从"双重职责"退回"纯画布显隐"，这是一次**行为迁移**。原来靠关眼睛来"不导出某图层"的用法，现在要改用"取消选择"。需在 UI 上给出区分（复选框 vs 眼睛图标）。

### 4.5 会话缓存（U6 IndexedDB）
新增的 `layerSelected` 状态要纳入快照，刷新/返回后保持。

### 4.6 重新上传 / 重新开始
重置时 `layerSelected` 与 `layerVisible` 一并清空重建（默认全选全显）。

| 5 | 默认值 | **全选 + 全显** |

## 6. UI 设计（第①步图层列表）

### 6.1 行结构变化

```
改造前：  [色块] 图层名 (数量) ............... 👁
改造后：  ☑ [色块] 图层名 (数量) ............. 👁
          ↑                                    ↑
        选择复选框（新）                    眼睛（纯画布，未选时置灰）
```

### 6.2 各状态的视觉

| 状态 | 复选框 | 行文字 | 眼睛 |
| :--- | :--- | :--- | :--- |
| 已选（默认）| ☑ | 正常 | 锁定常亮（淡灰锁态，`data-tip`="已选图层保持可见；要隐藏请先取消选择"）|
| 未选 + 可见（参考）| ☐ | 降为 `--ink-3` | 可点（睁眼）|
| 未选 + 隐藏 | ☐ | 降为 `--ink-3` | 可点（闭眼）|

### 6.3 顶部批量栏

图层列表标题行右侧，在现有「全显/全隐」眼睛旁，增加一个「全选/全不选纳入」的复选框或小按钮（沿用 ontotwin-ui 风格，无 emoji）。

### 6.4 统计提示（决策3=b 时）

step2 审核摘要、step4 实体区，增加一行灰字：
> "已按图层选择排除 N 项（在第①步可调整）"

## 7. 数据结构与改造点

### 7.1 前端 State 新增

```js
State.layerSelected = {};   // { 图层名: bool }，默认全 true
```

`applyPreviewData()` 初始化：`layers.forEach(l => { layerVisible[l]=true; layerSelected[l]=true; })`

### 7.2 改造函数清单

| 函数 | 改动 |
| :--- | :--- |
| `applyPreviewData` | 初始化 `layerSelected`（全 true）|
| `renderLayerList` | 行首加复选框；选中=锁眼睛常亮，未选=行置灰+眼睛解锁；加批量"全选/全不选纳入" |
| 新增 `toggleLayerSelect(name)` | 翻转 `layerSelected`；勾选时强制 `layerVisible=true`；重绘 + 刷新当前步 |
| `toggleLayer`（眼睛）| 仅在未选时生效；已选时无操作（眼睛锁定）|
| `draw()` | 过滤依据保持 `layerVisible`（不变）|
| `renderAuditList` | 候选过滤：跳过 `primary_layer` 未被选中的；摘要计数与"已排除"提示 |
| `buildEntityTable` | 过滤依据 `layerVisible` → `layerSelected` |
| `_buildExportPayload` | 同上 |
| `_collectSpawnItems`（2.9.1）| 同上 |
| `saveSession` / `restoreSession`（U6）| 快照纳入 `layerSelected` |
| `resetAll` | 清空 `layerSelected` |

### 7.3 后端

**无需改动**。scan 已返回 `primary_layer`，前端据此过滤即可（决策4=a）。
