# PROJECT_BRIEF.md — OntoTwin Lite 项目简报

## 一、一句话概括

> 为现有的 OntoTwin Nexus 数字孪生系统，新增一个"从本体定义导出 USD 训练场景"的能力，交付给具身智能训练侧同事使用。

---

## 二、为什么要做这个

### 背景

- 现有 OntoTwin Nexus 系统已经能做数字孪生的**本体建模**和**运行态可视化**（工人实时位置、AGV 轨迹等）
- 现在团队要把场景数据复用到**具身智能训练**，让机器人在仿真场景里学抓取、分拣
- 训练侧用的是 **NVIDIA Isaac Sim + Isaac Lab**，需要 **USD 格式**的场景
- 如果每次训练都让美术手工搭一遍场景，效率太低，且和业务数据脱节

### 目标

让场景配置成为"数据驱动"：
- 在 Nexus 前端配置场景（放哪些物体、在哪、多重、可抓取与否）
- 一键导出为 USD 文件
- 训练同事拿到 USD 直接在 Isaac Sim 加载训练

## 四、业务场景：仓储分拣

### 场景设定

一个仓库环境（地面 5m × 5m），场景中包含：
- **1 个 Franka 机械臂**（固定在桌面中央）
- **若干货架**（固定摆放，作为目标存放区）
- **若干箱子**（待抓取物体，初始化时随机放在机械臂工作区）

### 分拣规则

机器人要根据箱子**尺寸/质量**，把它们放到不同位置：

- **大箱子**（边长 ≥ 25cm 或质量 ≥ 1.5kg）→ 地面外侧货架
- **中箱子**（边长 15-25cm）→ 上层货架
- **小箱子**（边长 < 15cm）→ 回收篮

### 预计规模

- 一个场景
- 约 10 个物体（1 机器人 + 3 货架/容器 + 6-8 箱子）
- 几十个资产（FBX → USD）

**这是一个轻量级场景，不要按企业级规模设计。**

## 五、系统边界

### 本项目负责

- Nexus 前端新增"场景管理"页面
- Nexus 后端新增 USD 导出模块
- UE 工程（新建）加载导出的 USD 做可视化
- UE 里调整位置后可以回写到 Nexus
- FBX → USD 的资产转换
- 训练剧本文档

### 本项目不负责

- Isaac Sim 的安装和配置（训练同事自己搞）
- Isaac Lab 的训练代码（训练同事写）
- 奖励函数设计（训练同事定）
- 训练环境的随机化（训练同事做，可能会要求你导出时预留接口）

### 明确不做的事

- 不做多人协作
- 不做权限管理
- 不做版本控制（USD 文件直接覆盖，或手动备份）
- 不做云端部署（本地跑就行）
- 不做实时双向同步（只做文件级的导出 + 回写）
- 不做域随机化的完整实现（只在 USD 里预留字段）

## 六、关键决策速览

以下是已经和用户充分讨论后定下的决策，**Claude Code 不要再提问或质疑**：

| 决策项 | 选择 | 理由（简要）|
|---|---|---|
| 导出方向 | 单向（Nexus → USD）| 不做回流，避免复杂度 |
| USD 内容 | 静态场景 | 不含人员/AGV 的运行态数据 |
| 坐标单位 | 米 + Z-up + 右手系 | 和 Isaac Sim 一致 |
| UE 集成程度 | USD Stage 加载 + 位置回写 | 数据驱动，UE 是可视化端 |
| 资产格式 | FBX 转 USD 缓存 | 第一次用时转换并存 |
| 场景数量 | 1 个（第一版）| 规模小 |
| 后端存储 | SQLite 新库 + 保留 Nexus 现有存储 | 新旧隔离 |
| 异步任务 | 不做 | 同步执行，够快 |
| 训练框架 | Isaac Sim + Isaac Lab | 训练侧决定 |
| 训练任务 | 抓取 + 分拣 | 按尺寸分拣 |

详见 `ARCHITECTURE_DECISIONS.md`。

## 七、术语表

| 术语 | 含义 |
|---|---|
| OntoTwin | 本项目名，数字孪生平台 |
| Nexus | OntoTwin 的 Web 控制台 |
| Lite | 本次新增模块的代号，强调"轻量" |
| 本体（Ontology）| 描述业务对象类型和关系的图数据 |
| Instance（实例）| 本体里具体的某个对象（如"shelf_001"这个货架）|
| Scene（场景）| 一组 Instance 的空间组织，可导出为一个 USD 文件 |
| 能力接口（Interface）| 如 `I3D_Representable`，描述对象的三维能力 |
| USD | Universal Scene Description，Pixar 的开源 3D 场景格式 |
| Isaac Sim | NVIDIA 的机器人仿真平台 |
| Isaac Lab | 基于 Isaac Sim 的机器人学习框架 |
| VLA | Vision-Language-Action，视觉-语言-动作模型 |
| graspable | 可抓取物体（物理标记之一）|
