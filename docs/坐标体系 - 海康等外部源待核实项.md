# 坐标体系 — 外部坐标源待核实项（以海康为例）

> 背景：坐标统一方案把 ontotwin 设为「空间事实来源」（规范坐标系 canonical frame），
> 任意外部系统通过「定义一帧 + 一个到规范系的变换」接入。
> **海康不是特例，只是"外部坐标源"的一个样例（一类单位/坐标约定）。**
> 本版只做通用帧抽象，不写任何海康专属逻辑。下列问题等接入时现场核实，不阻塞本版。

---

## 原则

- 任何外部源（海康 RCS、其他 WMS/定位系统、人工 Excel…）= **一个源参考系 + 一个"源→规范"变换**。
- 接入新源 = 填一帧的参数，**不改架构**。
- 规范单位 = 毫米(mm)；外部源若是别的单位，在"源→规范"变换里做一次换算。

---

## 待核实项（海康样例）

### 1. 量纲/尺度是否一致
- 海康 `cooX/cooY` 是 mm 级平面坐标（样例 `cooX: 32408, cooY: 12227`）。
- **待核实**：同一物理点，海康报的 mm 数值 与 CAD 图上量的 mm 数值，是否对得上？
- **影响**：决定"海康→规范"是否只差平移+旋转（同尺度），还是还要带一个缩放系数。

### 2. mapCode 的粒度
- 样例 `mapCode: "DB"`。
- **待核实**：mapCode 是"一层楼一个"，还是"一层里按区域分多个"？一层会不会对应多个 mapCode？
- **影响**：`mapCode ↔ 楼层` 映射表是一对一还是多对一。

### 3. 原点与偏移
- 海康坐标系有 `originX / originY` 偏移。
- **待核实**：originX/Y 的确切含义与取值；是否每个 mapCode 一套。
- **影响**：作为"海康→规范"变换里的平移量。

### 4. 旋转/朝向
- **待核实**：海康平面坐标的 X/Y 朝向，与规范系（建筑系）朝向差多少角度、是否翻转。
- **影响**：作为"海康→规范"变换里的旋转/翻转。

### 5. Z / 楼层
- 海康是 2D 平面，无 Z。
- **待核实**：`mapCode → 楼层` 的对应，楼层 → Z 基准高度按建筑层高表取。

---

## 接入时怎么做（将来，非本版）

1. 在帧注册表新增一帧 `kind=external`，记 unit、mapCode、floor。
2. 标定"源→规范"变换：两种方式任选
   - **直接填参数**：originX/Y（平移）+ 尺度 + 旋转（海康若有文档化的坐标系参数）。
   - **锚点标定**：在两个系里各点 2~3 个已知同名点，最小二乘拟合仿射（海康不暴露参数时的退路）。
3. 实时数据进来后，`源坐标 →(源→规范)→ 规范坐标 →(规范→UE)→ UE`，全程矩阵运算。
